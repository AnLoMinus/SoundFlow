# 🤝 מדריך תרומה ל-SoundFlow

תודה על העניין שלך בתרומה ל-SoundFlow! 🎵

## 🎯 איך תוכל לתרום?

### 🐛 דיווח על באגים

אם מצאת באג:
1. בדוק אם הבאג כבר דווח ב-[Issues](https://github.com/AnLoMinus/SoundFlow/issues)
2. אם לא, פתח issue חדש עם:
   - תיאור מפורט של הבאג
   - צעדים לשחזור
   - דפדפן ומערכת הפעלה
   - צילומי מסך (אם רלוונטי)

### 💡 הצעות לשיפורים

אם יש לך רעיון לשיפור:
1. פתח issue עם התווית "enhancement"
2. תאר את הרעיון בפירוט
3. הסבר למה זה יעזור לפרויקט

### 📝 שיפור תיעוד

- תרגומים לשפות נוספות

### 🎨 שיפורי עיצוב

אם יש לך רעיונות לעיצוב:
- שיפור UI/UX

- שיפור נגישות

### 🎵 הוספת תוכן מוזיקלי

- הוספת כלי נגינה למאגר
- הוספת סגנונות מוזיקליים
- הוספת מצבי רוח
- יצירת שירים חדשים
- הוספת שיטות אפיון חדשות

### 🧪 הוספת בדיקות

## 🔄 תהליך התרומה

### 1. Fork את הפרויקט

לחץ על כפתור "Fork" ב-GitHub

### 2. צור Branch

```bash
git checkout -b feature/AmazingFeature
```

### 3. בצע שינויים

ערוך, הוסף, או מחק קבצים

### 4. Commit השינויים

```bash
git commit -m 'Add some AmazingFeature'
```

**הערה:** השתמש בהודעות commit ברורות ומתארות

### 5. Push ל-Branch

```bash
git push origin feature/AmazingFeature
```

### 6. פתח Pull Request

1. לך ל-GitHub של הפרויקט
2. לחץ על "New Pull Request"
3. בחר את ה-branch שלך
4. תאר את השינויים בפירוט
5. לחץ על "Create Pull Request"

## 📋 הנחיות לפיתוח

### סגנון קוד

- **HTML:** השתמש ב-semantic HTML
- **CSS:** עקוב אחר המבנה הקיים
- **JavaScript:** השתמש ב-ES6+ modern syntax
- **JSON:** שמור על עקביות בפורמט

### עריכת קבצים

- **כל קבצי הטקסט:** בעברית (RTL)
- **קוד:** תגובות בעברית, שמות משתנים באנגלית
- **תיעוד:** בעברית

### מבנה קבצים

כל שיר/אלבום צריך לכלול:
- `index.html` - עמוד ראשי
- `style.css` - עיצוב
- `script.js` - לוגיקה
- `metadata.json` - מטא-דאטה
- `README.md` - תיעוד
- `StableAudio_Prompt.txt` - פרומפט בעברית
- `StableAudio_Prompt_English.txt` - פרומפט באנגלית
- `YouTube_Description.txt` - תיאור ליוטיוב
- `Thumbnail_Description.txt` - הנחיות לעיצוב
